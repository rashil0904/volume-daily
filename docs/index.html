01 Strategy Overview

The pipeline targets NSE equities with market cap ₹1,500–5,000 Cr (small-to-mid cap). It screens for stocks where unusual volume has built through the day and the price has broken above the prior day's close — entering at the 3:15 PM market open with a defined capital budget.

02 Signal Logic

All three conditions must pass for a symbol to appear in the trade list. Evaluation happens at 3:00 PM using the candles present at that point.

✓
Market Cap
₹1,500 Cr < mcap < ₹5,000 Cr

Symbol must be present in today's Screener.in export. First gate — only tracked universe is evaluated.

✓
Volume
cum_vol(9:15–14:45) ≥ 6× avg_36d_vol

36-day rolling average of full-day volume (non-zero trading days only). Minimum 36 days of history required.

✓
Return
open(15:00) ≥ prev_vwap × 1.05

Prev-day VWAP from 15:00 + 15:15 candles: Σ((H+L+C)/3 × vol) / Σ(vol). Threshold: +5%.

03 Capital Allocation

Total capital is hard-capped at ₹5,00,000 regardless of how many signals fire. Shares per stock = floor(allocation / open_of_3pm_candle).

04 How It Works

run_pipeline.sh is triggered by cron at 3:01 PM IST and calls pipeline/run_daily.py, which runs these steps in sequence:

1
Fetch Market Cap fetch_market_cap.py

Logs into Screener.in Premium, runs the query Market Capitalization > 1500 AND Market Capitalization < 5000, and exports today's symbol list to data/market_cap_daily/market_cap_<date>.csv. If Screener is unavailable, falls back to the most recent previous export and continues with a warning.

2
Update Universe run_daily.py

Compares today's market-cap symbols against data/universe_combined.csv. Any symbols not previously tracked are appended automatically — the universe expands each day as new stocks enter the cap band.

3
Candle Data Update fetch_candles.py

Two sub-operations: (a) new symbols — downloads the Upstox NSE instrument master, matches symbols to instrument keys, backfills 1 year of 15-min candles; (b) all symbols — fetches today's intraday 15-min candles and appends new rows to every candle file. 5 parallel workers · 0.8s call delay · ~66 req/min.

4
Generate Trade List prepare_data.py

Applies all three signal conditions to every symbol with both a candle file and a current market-cap record. Passing symbols are written to results/trade_list_<date>.csv (columns: symbol, shares, ref_price). If no signals fire, no file is written.

✉
Telegram Notification notify.py

Sends a message to the NSE Volume Alerts Telegram group. Success: full trade table with symbol, shares, and entry price. Failure: failed step name and error detail. Notification errors are logged as warnings and never crash the pipeline.

06 First-Time VM Setup

Prerequisites: DigitalOcean Ubuntu 22.04+ droplet with SSH access. The GitHub repo must be accessible from the VM (deploy key or HTTPS).

bash
# 1. SSH into the droplet
ssh root@<DROPLET_IP>

# 2. Clone the repo
git clone git@github.com:rashil0904/volume-daily.git
cd volume-daily

# 3. Run the setup script (installs Python, sets IST timezone, wires up cron)
chmod +x setup_vm.sh
./setup_vm.sh

# 4. Fill in credentials
nano pipeline/.env

# 5. Verify everything works
python3.11 pipeline/run_daily.py

setup_vm.sh handles automatically:

a
Installs python3.11, python3.11-venv, git
b
Sets system timezone: timedatectl set-timezone Asia/Kolkata
c
Installs Python dependencies from pipeline/requirements.txt
d
Creates data/candles, data/instruments, data/market_cap_daily, results
e
Writes a blank pipeline/.env (if not already present)
f
Registers cron: 1 15 * * 1-5 /bin/bash ~/volume-daily/run_pipeline.sh >> ~/pipeline.log 2>&1

08 Daily Operations

Token Refresh

A data token and a trading token, with different lifecycles, both need to be valid for the pipeline to run end to end:

Candle Data Token
UPSTOX_ACCESS_TOKEN

Long-lived token from the Upstox developer portal. Used by fetch_candles.py to pull OHLCV data. Update in pipeline/.env when it eventually expires. Analytics only — no order placement.

Zerodha Trading Token
zerodha/.token.json

Expires at 6:00 AM IST every day (Kite Connect regulatory requirement). Refresh each morning before trading. Saved automatically after OAuth — no manual file editing needed.

refresh trading token
python -m zerodha.auth

09 Manual Commands

cheat sheet
# Run the full pipeline manually
python3.11 pipeline/run_daily.py

# Preview today's trades — no orders placed
python zerodha/execute_trades.py --dry-run

# Execute today's trades (live)
python zerodha/execute_trades.py

# Execute for a specific date
python zerodha/execute_trades.py --date 2026-07-17

# Test Telegram — success
python pipeline/notify.py --date 2026-07-17 --status success

# Test Telegram — failure
python pipeline/notify.py --date 2026-07-17 --status failed \
  --failed-step "Step 3" --error-msg "API timeout"

# Manual single order
python -m zerodha.trade buy  RELIANCE NSE 10 MARKET
python -m zerodha.trade sell RELIANCE NSE 10 MARKET

# Order management
python -m zerodha.trade orders               # all today's orders
python -m zerodha.trade status <order_id>    # single order status
python -m zerodha.trade cancel <order_id>    # cancel an order

# Auth
python -m zerodha.auth         # daily login + save token

10 Telegram Notifications

Notifications go to the NSE Volume Alerts Telegram group via @nse_volume_alerts_bot.

A success message includes: date, signal count, pipeline runtime, and a monospaced trade table with symbol · shares · entry price. A stale market-cap warning is prepended if Screener.in data is from a previous day.

A failure message includes: date, failed step name, runtime, and the full error string.

Setting Up on a New Bot or Group

1
Create a bot via @BotFather → copy the token → set TELEGRAM_BOT_TOKEN in pipeline/.env
2
Add the bot to your Telegram group
3
Send any message in the group, then call getUpdates to retrieve the chat ID
4
Set TELEGRAM_CHAT_ID to the negative integer returned (supergroups always have negative IDs)
get chat id
